## Architecture

This is a **Cloudflare Workers** app that hosts a remote MCP (Model Context Protocol) server using the `agents` SDK. No authentication is required.

### Key concepts

- **`McpAgent`** (from `agents/mcp`): The base class for the MCP server. `MyMCP` extends it and is backed by a **Cloudflare Durable Object** with SQLite storage (configured in `wrangler.jsonc` under `durable_objects` and `migrations`).
- **`McpServer`** (from `@modelcontextprotocol/sdk`): Registered as `this.server` inside `McpAgent`. Tools are defined in the `init()` method using `this.server.tool(name, zodSchema, handler)`.
- The Worker's `fetch` handler in `src/index.ts` routes `/mcp` to `MyMCP.serve("/mcp")`. Any other path returns 404.
- The MCP endpoint is `/mcp` (not `/sse` — note the README mentions `/sse` but the actual code uses `/mcp`).

### Adding tools

All tools are defined inside `MyMCP.init()` in [src/index.ts](src/index.ts). Use `this.server.tool()` with a Zod schema for input validation. Tool handlers return `{ content: [{ type: "text", text: "..." }] }`.

### Bindings

`worker-configuration.d.ts` is auto-generated by `npm run cf-typegen` and defines the `Env` interface from wrangler bindings. Do not edit it manually.
